# v1.0.0

## 魔方无纸化工具
> 这是一个用于魔方最少步的输入工具, 用于最少步能够支持便捷输入，实现无纸即可方便快捷的进行最少步练习。


## 研发技术
- 这是一个纯前端项目
- 使用react + ts编写程序

## 整体风格
- 风格整体为淡蓝色调。
- 圆角风格。

## 功能
本工具在本期研发中需要支持以下功能:
- 输入或者自动生成打乱。
- 便捷键盘。
- 最少步链。
- 影子解法。
- 注释
- 插入
- 完整解法整理。
- 解法标记。
- 手动整理板。
- 导入导出和本地缓存。
- 计时器


## 功能详解
1. 输入和自动生成打乱
    - https://www.npmjs.com/package/cstimer_module
    - 使用cstimer module来展示图片和自动生成打乱。
    - 打乱需要一直在最顶部。
    - 打乱图可隐藏或者在左上角点击显示。
2. 虚拟键盘
  - 能够支持:
    - R L U D F B
    - Rw Lw Uw Dw Fw Bw / x y z 等
    - 能够支持 `'` `2` `w`等后缀， 如： R' \ Rw' \ x', R2 \ Rw2 \ x2
    - 能够支持删除, 能够支持左右括号
    - 注意， `w` 后缀不能用于x y z 后续
  - 默认布局为4层的输入:
    - R L U D F B
    - R' L' U' D' F' B'
    - x y z  `2` `'` `w`
    - `(` `)` `插入#` `<删除`
  - 可支持的动态布局行
    - Rw Lw Uw Dw Fw Bw
    - Rw' Lw' Uw' Dw' Fw' Bw'
    - 手动配置，支持手动切换。
    - 最多可配置6行的虚拟键盘，用户可自由添加或者移除。
  - 连续点击
    - 连续点击两个R的时候，会变为R2
    - 连续点击三个R的时候，变成成R'
    - 连续点击四个R的时候，会直接为空
    - 连续点击两个R'的时候，变成R2, 3个会变成R
      - 以此类推
  - 长按
    - 长按时，会在按钮顶部弹出三种不同的可点击按钮
    - 比如R： R R' R2
    - 支持滑动切换。
  - 位置
    - 默认常驻于最下面
    - 支持切换为最左边、最右边、最下边，自由悬浮。
  - 虚拟键盘可关闭，用户可自己输出。
3. 最少步链
   - 最少步链是指一个最少步解法会存在多个步骤
   - 例如: 
   ```text
    D L B' U  B' L'            // EO      4/6
    B F' L' B' L'              // 4A1     5/11
    R2 U2 L2 U2 L2 F           // HTR     6/17
    D2 L2 U2 B' F' U2 F B'     // FINISH  8/25
    ```
    这里的 EO 4A1 HTR FINSIH即是不同的步骤
   - 用户输入一个前置解法后，需要可分支化，类似于思维导图，即树状分布。
   - 用户可自由在某个分支后面追加解法、或者做兄弟解法。
4. 注释
   - 用户可以给最少步链上任意的解法添加一个步骤注释。
   - 注释长度、内容不限
5. 影子解法
   - 比如用户写了一个EO后， 为: R U R'
     - 则在最少步链中， 可点击添加影子解法，即在这个解法添加一个兄弟解法，把最后一步改为 R U R
       - 即 A 改为 A', B' 改为 B
       - 但是A2 不需要修改
6. 插入
   - 插入可在最少步链的任意位置插入一个子解法。
   - 子解法附属于这个最少步链的外围，可以理解为这个最少步链添加一个模块化的公式。
   - 如:
   ```text
    (R B' L U F) // EO
    D' // JZP
    (U2 # L2 B2 U R2 D) // DR 4b2 2e (12)
    (F2 U2 L2 F2 L' B2 R) // HTR (19)
    (B2 U2 L2) // 2e2e (22)
   
    # = U2 B2 U2 L2 B2 U2 B2 L2 // (8-1)
   ```
    这里的#即是一个子解法，在解法当中，可以直接插入进去，即`(U2 U2 B2 U2 L2 B2 U2 B2 L2 L2 B2 U R2 D)`
   - 允许使用相同的插入，比如可以在多个位置插入同一组公式。
   - 允许 w 插入
    ```text
    (D2 F R)
    B' D (B R2 B2 D') //9 2c4
    (F') U2 F#' U2 F' U2 F#'
    R2 D2 F^2 R2 U2
   
    # = w
    ```
    即插入可以是`w`或者`w2`， 或者`'`, 这里`U2 F#' U2 F' U2 F#'` 插入w后就变成了`U2 Fw' U2 F' U2 Fw'`

7. 完整解法整理
   - 在解法当中正序解法需要把最少步链整理完成，然后输出一个正确的解法
   例如
   ```text
    (B2 U F' L)//EO
    (F D' U2 F B R2 F2 U)//DR
    F R2 F D2 R2 F U2 F' //HTR
    (R2 U2 R2 D2 F2 )//FINISH
    ```
   需要把正序(没有带括号的解法放出来)， `F R2 F D2 R2 F U2 F'`
   然后在把括号的内容反向写上，即 `F2 D2  R2 U2 R2 U' F2 R2 B' F' U2 D F' L' F U' B`
   - 然后合并解法时，需要注意消步 `F R2 F D2 R2 F U2 F' F2 D2  R2 U2 R2 U' F2 R2 B' F' U2 D F' L' F U' B`
     - 这里F' 和 F2 可以合并为 F, 即最终输出为`F R2 F D2 R2 F U2 F D2  R2 U2 R2 U' F2 R2 B' F' U2 D F' L' F U' B`
   - 同时注意，如果存在`U' D2 U'`, 则也可以合并`U2 D2`， 即如果`UD`, `FB`, `LR` 的连续步骤，可以进行合并动作。
     - 但是注意， Uw Dw和 U D无法作为一组，即带w的只能和带w的消步
8. 最少步链中， 可以给每个解法做单独的标记优先级，或者设置特殊的背景。
9. 手动整理板
   - 除了最少步链模块， 可以单独配置一个手动整理板子
   - 这个板子可以直接添加最少步链中的任意一套解法的文字版本进来。按行写节点的
   - 参考上文提供的例子即可。
10. 计时器
    - 计时器默认60分钟的计时。
    - 计时器默认正数，支持倒数
    - 1秒一跳
    - 当不足15分钟时，跳为橙黄色
      - 不足5分钟时，时间背景为红色
    - 时间以时间戳为准。
    - 超时时可继续编写。
11. 导入导出和本地缓存
   - 本地可缓存每一把解法包含过程。
   - 导出时按照时间（详细到秒）的json文件
   - 本地缓存时，自动恢复到最新的一次解法。
